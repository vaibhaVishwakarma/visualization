Complexity Analysis

Core Logic:
Even if an array is rotated, at least one half of the array (either s to mid, or mid to e) will always be perfectly sorted. We can check if the target lies within this sorted half. If it does, we discard the other half. If not, we discard the sorted half.

Time Complexity: O(log N)
The search space [s, e] is halved in every step, leading to logarithmic time complexity.

Space Complexity: O(1)
Only a few variables (s, e, mid, n) are used, making it an in-place algorithm.
