JAVA CODE (BINARY SEARCH ON PARTITION)


Complexity Analysis

Core Logic:
Instead of merging the arrays, we find a "partition" in both arrays such that the total number of elements on the left side equals the total number on the right. By binary searching the partition index in the smaller array, we instantly find the matching partition in the larger array.

Time Complexity: O(log(min(M, N)))
We only perform binary search on the smaller array (let's say of size M). The search space halves every iteration.

Space Complexity: O(1)
We only use primitive pointer variables (`low`, `high`, `partitionX`, etc.) meaning memory usage is constant.
